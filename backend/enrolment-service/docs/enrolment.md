# Agent Enrolment & Identity (`api/`, `application/`, `domain/`, `infrastructure/`)

Handles the secure generation of bootstrap credentials, initial connection verification, and persistent cryptographic binding of new endpoints.

### code
**API & Routing**
[`EnrolmentController.java`](../src/main/java/dev/pulsermm/enrolment/api/controller/EnrolmentController.java):
	- `generateToken` & `getEndpoints` - Expose REST endpoints allowing administrators to create new registration tokens and query the fleet of successfully enrolled agents.

**Data Transfer Objects (DTOs)**
[`CreateTokenRequest.java`](../src/main/java/dev/pulsermm/enrolment/api/dto/CreateTokenRequest.java) & [`TokenResponse.java`](../src/main/java/dev/pulsermm/enrolment/api/dto/TokenResponse.java):
	- Encapsulate the inbound constraints (like token lifespan or allowed uses) and the outbound generated secret utilized for bootstrapping new machines.
[`EndpointResponse.java`](../src/main/java/dev/pulsermm/enrolment/api/dto/EndpointResponse.java):
	- Structures the public profile of a registered endpoint, masking internal database identifiers while exposing the UUID and system hostname.

**Domain Exceptions**
[`InvalidTokenException.java`](../src/main/java/dev/pulsermm/enrolment/api/errors/InvalidTokenException.java):
	- Thrown during the bootstrap phase if an agent presents a token that does not exist, has expired, or has exhausted its maximum permitted uses, actively refusing the registration attempt.

**Application Logic**
[`TokenService.java`](../src/main/java/dev/pulsermm/enrolment/application/TokenService.java):
	- `createEnrolmentToken` & `validateToken` - Manage the cryptographic generation of short-lived secrets and execute strict validation checks before allowing an enrolment to proceed.
[`EnrolService.java`](../src/main/java/dev/pulsermm/enrolment/application/EnrolService.java):
	- `enrolEndpoint` - Orchestrates the entire bootstrap workflow, verifying the submitted token, generating a permanent system UUID, and persisting the hardware's public key to establish a permanent trust anchor.

**Domain Entities**
[`EnrolmentToken.java`](../src/main/java/dev/pulsermm/enrolment/domain/EnrolmentToken.java):
	- Represents a short-lived, cryptographically secure string generated by administrators, acting as the initial proof of authorization for a new machine.
[`Endpoint.java`](../src/main/java/dev/pulsermm/enrolment/domain/Endpoint.java):
	- The core domain entity representing a successfully registered machine, storing its UUID, hostname, public key, and its current position within the organizational hierarchy.

**Infrastructure & Integrations**
[`EnrolmentTokenRepository.java`](../src/main/java/dev/pulsermm/enrolment/infrastructure/EnrolmentTokenRepository.java) & [`EndpointRepository.java`](../src/main/java/dev/pulsermm/enrolment/infrastructure/EndpointRepository.java):
	- Interface directly with the database to persist and query token validity and retrieve registered endpoints by their unique identifiers.
[`EnrolmentGrpcService.java`](../src/main/java/dev/pulsermm/enrolment/infrastructure/grpc/EnrolmentGrpcService.java):
	- Acts as the receiving end of the internal gRPC proxy network, accepting initial bootstrap payloads from raw agents connecting through the API Gateway.
[`IdentityServiceClient.java`](../src/main/java/dev/pulsermm/enrolment/infrastructure/IdentityServiceClient.java):
	- Synchronizes domain state across microservices by notifying the Identity Service when new groups are created, ensuring Role-Based Access Control scopes remain accurate.

### description
When the Go agent is installed on a new device, it possesses no identity and no trust relationship with the backend. To bridge this gap, an administrator uses the frontend application to hit the `EnrolmentController`, which relies on the `TokenService` to generate a secure `EnrolmentToken` entity. The administrator passes this raw token string to the physical endpoint during installation. 

The agent then initiates an outbound connection, sending its freshly generated public key alongside the token. This payload routes through the API Gateway and lands in the `EnrolmentGrpcService`. The payload is handed off to the `EnrolService`. This core service queries the `EnrolmentTokenRepository` to confirm the token is valid and hasn't been exhausted, potentially throwing an `InvalidTokenException` if tampering is detected. Once authorized, the service generates a permanent UUID for the new machine. It persists an `Endpoint` entity into the database, inextricably linking the newly generated UUID to the hardware's public key. This strict cryptographic binding ensures that all future commands routed to this UUID are verifiable, preventing bad actors from impersonating endpoints even if they discover internal network addresses.
