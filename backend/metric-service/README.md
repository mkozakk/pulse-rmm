# Metric Service (`backend/metric-service/`)

The Metric Service acts as the ingestion engine for all telemetry data generated by the endpoint fleet. It processes high-frequency heartbeats and hardware metrics, storing them efficiently for historical analysis, real-time dashboarding, and endpoint connectivity monitoring.

## Directory Structure
```text
metric-service/
├── docs/                   # Detailed feature documentation
├── src/main/java/.../
│   ├── api/                # HTTP Controllers, Security Filters, and OpenAPI configuration
│   ├── application/        # Ingestion logic and offline detection background jobs
│   ├── domain/             # Heartbeat and Metric entities
│   └── infrastructure/     # TimescaleDB repositories and internal gRPC ingestion points
├── Dockerfile              # Containerization definition
└── pom.xml                 # Maven dependencies
```

## Features & Internal Documentation

* **[Telemetry Ingestion](docs/ingestion.md)** - Details how high-volume hardware metrics and heartbeats are received via gRPC, normalized, and persisted to TimescaleDB.
* **[Status Monitoring](docs/status.md)** - Explains the asynchronous mechanisms used to determine if an endpoint has lost network connectivity.
* **[API & Configuration](docs/config.md)** - Covers the public data retrieval endpoints, internal security boundaries, and application bootstrap.
