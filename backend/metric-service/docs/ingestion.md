# Telemetry Ingestion (`application/`, `domain/`, `infrastructure/`)

Handles the high-volume intake, normalization, and highly optimized storage of agent telemetry and hardware performance data.

### code
**gRPC Ingestion Endpoints**
[`HeartbeatGrpcService.java`](../src/main/java/dev/pulsermm/metric/infrastructure/grpc/HeartbeatGrpcService.java) & [`MetricGrpcService.java`](../src/main/java/dev/pulsermm/metric/infrastructure/grpc/MetricGrpcService.java):
	- Act as the primary high-speed receivers for Protobuf data streams originating from the agents. They intercept incoming hardware statistics (CPU, RAM, Disk) and lightweight connectivity pings routed through the API Gateway, ensuring rapid network transfer with minimal overhead.

**Application Logic**
[`MetricIngestionService.java`](../src/main/java/dev/pulsermm/metric/application/MetricIngestionService.java):
	- Validates incoming telemetry payloads, normalizes the data structures, and orchestrates the batching process to prepare the high-velocity streams for efficient database insertion.

**Domain Entities**
[`EndpointHeartbeat.java`](../src/main/java/dev/pulsermm/metric/domain/EndpointHeartbeat.java):
	- A lightweight domain entity representing a single, timestamped connectivity ping from an active agent, explicitly tying a specific point in time to an endpoint's UUID.

**Infrastructure Repositories**
[`EndpointHeartbeatRepository.java`](../src/main/java/dev/pulsermm/metric/infrastructure/EndpointHeartbeatRepository.java):
	- Interfaces directly with the PostgreSQL database (augmented by the TimescaleDB extension) to append the time-series heartbeat data into highly optimized, time-partitioned hypertables.

### description
Because physical agents report their operational status and hardware utilization constantly, the infrastructure must process an immense volume of data without bottlenecking or degrading database performance. Standard REST HTTP overhead is too heavy for this continuous task. Instead, the Metric Service exposes raw gRPC ingestion points via the `MetricGrpcService` and `HeartbeatGrpcService`. The API Gateway simply proxies the raw network streams directly from the physical agents to this internal module. 

Once the data arrives, the `MetricIngestionService` sanitizes and restructures the payloads. Crucially, standard relational database tables degrade in performance when confronted with millions of timestamped rows generated by a large fleet. To solve this, the infrastructure relies on TimescaleDB. The repositories (like `EndpointHeartbeatRepository`) interact with logical abstractions called hypertables. Behind the scenes, these hypertables automatically partition the incoming `EndpointHeartbeat` entities by time intervals, allowing the database to maintain incredibly fast insert speeds while simultaneously supporting the complex, time-based aggregate queries required by the frontend web application dashboards.
