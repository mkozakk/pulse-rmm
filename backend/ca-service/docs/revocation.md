# Revocation & Certificate Revocation List (CRL)

How certificate revocation works and how agent-hub validates certificates against the revocation list.

## Revocation Overview

A revoked certificate can no longer be used to establish new connections. Revocation is irreversible and permanent for the lifetime of that certificate (until it expires, at which point it's no longer valid anyway).

**Use cases:**
- An endpoint is decommissioned (endpoint retired, VM deleted)
- An agent's private key is compromised (certificate stolen)
- An agent should be temporarily offline (operator intervention)

## Certificate Revocation List (CRL)

The CRL is a signed document that lists the serial numbers of revoked certificates. It's generated by ca-service and distributed to agent-hub.

**CRL Format:**
- X.509 v2 CRL (standard format)
- Signed by the root CA private key
- Contains list of revoked certificate serial numbers
- Includes nextUpdate timestamp (when the CRL should be refreshed)

**CRL Content Example:**
```
Certificate Revocation List (CRL):
    Signature Algorithm: sha256WithRSAEncryption
    Issuer: CN = Pulse RMM CA
    thisUpdate: Jan  1 00:00:00 2026 GMT
    nextUpdate: Feb  1 00:00:00 2026 GMT
    Revoked Certificates:
        Serial Number: 0x1a2b3c4d5e6f7g8h
            Revocation Date: Jan 15 10:30:00 2026 GMT
        Serial Number: 0x9h8g7f6e5d4c3b2a
            Revocation Date: Jan 20 14:15:00 2026 GMT
```

## How Agent-Hub Uses the CRL

**On Startup:**
1. Agent-hub fetches the CRL from ca-service: `GET /api/ca/crl`
2. Validates the CRL signature (signed by root CA)
3. Parses the CRL and loads revoked serial numbers into memory
4. Sets a timer to refresh the CRL every 1 hour

**On Each Agent Connection:**
1. Agent presents a certificate (mTLS)
2. Agent-hub validates the certificate:
   - Signature is valid (signed by root CA)
   - Not expired
   - Subject CN is a valid UUID
3. Agent-hub checks the CRL:
   - Extract certificate serial number
   - Look up serial in revoked list (in-memory map)
   - If found: Reject connection with `UNAUTHENTICATED` status
   - If not found: Accept connection

**Periodic Refresh:**
Every 1 hour, agent-hub fetches the latest CRL to pick up newly revoked endpoints.

## Revoking an Endpoint

There is no public API to revoke an endpoint in the current implementation. Revocation is done directly in the database:

```sql
-- In ca_service database, schema ca:
INSERT INTO certificate_revocation_list (serial_number, revoked_at, reason)
VALUES ('0x1a2b3c4d5e6f7g8h', NOW(), 'ENDPOINT_DECOMMISSIONED');
```

In production, this would typically be done via:
1. A private internal API: `DELETE /internal/agents/{endpointId}` (marked as revoked)
2. Or an operator command: `pulse-cli endpoint revoke <endpoint-uuid>`

## Revocation Latency

There's a latency window between revocation and rejection:

1. **Immediate:** Newly revoked endpoints are added to the CRL
2. **Within 1 hour:** Agent-hub refreshes its local CRL, rejects the revoked endpoint
3. **Existing connections:** If an agent was already connected before revocation, it remains connected until:
   - Network loss (agent reconnects, sees it's revoked, can't reconnect)
   - Explicit stream termination by agent-hub (future feature)

For critical revocation (e.g., compromised key), the operator should also terminate the agent's stream directly (not currently exposed in the API).

## Implementation

**CaService (`application/CaService.java`):**
- `getCrl()` — Fetches current revocation list, generates new CRL if needed
- `isRevoked(serialNumber)` — Checks if a certificate serial is revoked
- Revocation list is stored in the database (`certificate_revocation_list` table)

**Agent-Hub (`infrastructure/grpc/MtlsInterceptor.java`):**
- `validateCertificate(clientCert)` — Checks certificate and CRL
- Maintains in-memory CRL snapshot with periodic refresh
- Returns error if certificate is revoked

## Database Schema

**ca_service, schema ca:**

```sql
CREATE TABLE certificate_revocation_list (
    serial_number VARCHAR(255) PRIMARY KEY,
    revoked_at TIMESTAMP NOT NULL,
    reason VARCHAR(100),  -- ENDPOINT_DECOMMISSIONED, COMPROMISED, etc.
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**agent-hub (in memory only):**
- `Set<String> revokedSerials` — Snapshot of revoked certificate serials
- Refreshed every 1 hour by calling ca-service

## Operational Scenarios

### Scenario 1: Endpoint Decommissioned

1. Operator runs: `pulse-cli endpoint delete <uuid>`
2. Endpoint-service marks endpoint as inactive
3. CA service adds certificate serial to revocation list
4. Agent tries to reconnect, CRL check fails
5. Agent goes offline permanently (can't reach hub)
6. After 1 hour, new agent connections fail CRL check

### Scenario 2: Private Key Compromised

1. Operator suspects agent private key was stolen
2. Operator runs: `pulse-cli endpoint revoke <uuid> --reason=COMPROMISED`
3. CA service immediately revokes the certificate
4. Agent tries to reconnect, CRL check fails
5. Agent is locked out
6. Operator re-enrolls the endpoint to get a new certificate

### Scenario 3: Bulk Revocation (e.g., during breach)

1. Operator runs: `pulse-cli endpoint revoke-all --reason=SECURITY_INCIDENT`
2. CA service revokes all endpoint certificates
3. All agents trying to reconnect fail CRL check
4. System is locked down
5. Operator re-enrolls all endpoints one by one or in bulk

## Limitations & Future Improvements

**Current limitations:**
- No API to revoke an endpoint (direct database update needed)
- CRL refresh is every 1 hour (no real-time revocation notification)
- Existing connections are not terminated (only new connections are rejected)

**Future improvements:**
- Public API: `DELETE /api/endpoints/{id}` to revoke
- Real-time revocation: Push notification to all hub instances when CRL changes
- Graceful disconnect: Send command to agent before CRL check fails
- OCSP (Online Certificate Status Protocol): More efficient than CRL for large deployments

## Security Considerations

**CRL Integrity:** The CRL is signed by the root CA, so tampering is detectable.

**CRL Availability:** If ca-service is unavailable, agent-hub uses a cached CRL. If the cache is stale (older than `nextUpdate`), new connections may be rejected conservatively.

**Revocation Timing:** There's a window between revocation and enforcement (up to 1 hour). For emergency revocation, operator should manually terminate streams or use an out-of-band mechanism.
